## Honest Feedback
1. **GitHub Deploy Latency**: The 30-40 second wait time for GitHub Actions `pages-build-deployment` adds significant friction. It necessitates hard refreshes and creates a disjointed feedback loop when troubleshooting live visual errors.
2. **Material Debugging Complexity**: Tuning WebGL materials (Standard vs. Physical vs. Basic) blindly without a visual inspector relies heavily on trial, error, and deep framework knowledge. Ambient light bleeding heavily skewed the perceived hex colors of our data nodes.
3. **Submodule Surprises**: Cloning an external git repository (KlakMath) into the workspace silently introduced an embedded repository. This broke the deployment pipeline unexpectedly, requiring an emergency `git rm --cached` intervention to unblock progress.

## Lessons Learned
When building data visualizations in a 3D environment, the choice of material dictates data integrity. If a node *must* accurately display a specific UI color (like a hex code mapping to an AQI level), do not use `MeshStandardMaterial` or `MeshPhysicalMaterial` in a dark scene. The ambient lighting will heavily darken or wash out the color. Instead, use `MeshBasicMaterial`, which is unlit and guarantees the exact pixel color rendering, making neon data nodes "pop" vibrantly regardless of the room's darkness.
